# MQTT Telemetry Client for Embedded Systems

Designed and implemented an MQTT Telemetry Client in C for embedded systems to simulate publish-subscribe communication and real-time telemetry transmission. Developed core modules including MQTT Client Manager, Connection Handler, Publisher, Subscriber, Topic Manager, and Packet Encoder/Decoder. Verified functionality through software simulation and message analysis.


## Overview

Designed and implemented a lightweight MQTT telemetry client inspired by IoT and industrial embedded applications. The project focuses on understanding MQTT communication, topic management, packet handling, and publish-subscribe mechanisms. The framework demonstrates reliable telemetry transmission between embedded devices and cloud applications.


## Project Objectives

* Understand MQTT protocol architecture and operation.
* Implement publish-subscribe communication mechanisms.
* Simulate telemetry data transmission and reception.
* Develop topic management and packet handling capabilities.
* Gain practical experience with IoT communication protocols.

## System Architecture

The MQTT Telemetry Client consists of connection management, topic handling, publishing, subscription, and packet processing modules. Telemetry data generated by the application layer is encoded and transmitted to MQTT brokers while incoming messages are decoded and routed to subscribers.

## Core Modules

### MQTT Client Manager

Coordinates MQTT operations and session management.

### Connection Handler

Establishes and maintains communication with the MQTT broker.

### Publisher Module

Handles telemetry data publishing to configured topics.

### Subscriber Module

Receives and processes subscribed messages.

### Topic Manager

Manages topic registration and subscription lists.

### Packet Encoder

Constructs MQTT packets for transmission.

### Packet Decoder

Parses incoming MQTT packets and extracts payload information.

### Event Logger

Records connection events and communication statistics.
